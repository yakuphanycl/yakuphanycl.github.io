A Practical Coordination Protocol for Concurrent AI Coding Agents

When multiple AI agents (Claude Code, Codex, Cursor) work simultaneously on a shared codebase, git only handles committed state. This paper describes a three-layer protocol — runtime intent registry, durable file locks, and SSE broadcast — that fills the coordination gap from task inception to first commit.

Problem: AI agents have no visibility into each other's in-progress work.
Contribution: Conflict taxonomy (Type I–IV) + minimal CLI-based coordination protocol.
Implementation: wrg-agent CLI + Control Center SSE + pre-commit zorunluluğu.

Problem: Concurrent AI agents (Claude Code + Codex) had no visibility into each other's in-progress work — conflicts surfaced only at merge time.

Solution: Three-layer ACP: SQLite TTL locks (runtime), .agents/ JSON files (durable + auditable), SSE broadcast (real-time). 8-command CLI, pre-commit hook enforcement.

Outcome: Prevented duplicate work, structural and intent conflicts across 3 real multi-agent sessions. Protocol published as ACP_PAPER.md.
